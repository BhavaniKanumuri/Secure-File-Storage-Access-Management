# Secure File Storage and Access Management for Project Teams

## Project Summary:
Designed and implemented a secure file storage and access management solution for project teams on Linux. Configured Role-Based Access Control (RBAC), Access Control Lists (ACLs), file ownership permissions, and sticky bit policies to protect shared resources from unauthorized access, modification, and deletion. Deployed auditd for continuous monitoring of file access, permission changes, and user activities, supporting security auditing and forensic investigations. Integrated an Apache2-based dashboard to provide centralized visibility into security logs, alerts, and compliance reports, enhancing security monitoring and governance.

## Tools Used:
Linux Command-Line and ACLs, Shell Configuration, HTML Reporting

## Implementation Steps:
- Implemented a secure Linux-based file storage and access management solution, configuring Role-Based Access Control (RBAC), Access Control Lists (ACLs), file ownership, and sticky bit policies to prevent unauthorized access, modification, and deletion of shared resources.
- Deployed `auditd` to monitor file access, permission changes, and user activities, enabling detection of unauthorized actions and supporting security auditing, compliance, and forensic investigations.
- Integrated an Apache2-based secure web dashboard for centralized security monitoring, providing real-time visibility into logs, alerts, and compliance reports to improve security governance.
